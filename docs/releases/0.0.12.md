# 0.0.12

## Nowości
- Nowy backend audio na BASS-ie z pełnym wsparciem dla ASIO: każde odtworzenie startuje na świeżo, tryb loop działa także na ASIO, a pakiet Windows zawiera komplet 64-bitowych bibliotek BASS i bs1770gain.
- Automix przeszedł gruntowny lifting – kolejka Play Next zawsze idzie sekwencyjnie, breaki mają pamięć wznowienia, a mix preview w PFL korzysta z tych samych wyliczeń (loop crossfade, fade-in, overlap).
- Wprowadzono dokładniejsze okna guard dla natywnych miksów: kiedy backend nie zdąży, SARA odpala fallback progresowy w czasie zależnym od brakującego zapasu, co gwarantuje pełne wykorzystanie dostępnego fadera.
- Dodano panel diagnostyczny i logowanie per sesja z poziomu UI: start aplikacji zapisuje logi do `logs/` i można przełączać poziom głośności logów z okna opcji.
- Dodatek NVDA zyskał tryb „Nie przeszkadzać” (`NVDA+Alt+D`/gestury laptopowe), dokładniejsze wykrywanie ręcznej nawigacji oraz bezpośrednie emulowanie `NVDA+Tab`, więc playlisty mówią tylko wtedy, gdy użytkownik rzeczywiście nimi steruje.

## Poprawki
- Zatrzymanie/uruchamianie utworu nie psuje selekcji, kiedy `follow-playing` jest wyłączone; przełączenia loop/swap play/select zachowują fokus i wcześniejsze zaznaczenia.
- Automix respektuje breaki i pętle: break natychmiast zatrzymuje mix, a utwory z aktywnym loopem blokują ponowne zaplanowanie triggera.
- Tryb manualnego fade’u wyłącza automix konsekwentnie, a guardy natywnego miksu reagują na brak zapasu zarówno przy krótkich fade’ach, jak i przy długich crossfade’ach.
- Dodatek NVDA ignoruje automatyczne zaznaczenia, skraca okno mowy przy Play Next i traktuje ogłoszenia z prefiksem jako już wypowiedziane, dzięki czemu nie ma podwójnego gadania.

## Znane uwagi
- Budowanie wxPython lokalnie nadal wymaga GTK (Linux) lub Visual Studio Build Tools (Windows).
- Pełne wsparcie ASIO/WASAPI/Qt pozostaje w extrasach: `pip install -e ".[dev,asio,wasapi,qt]"`.
